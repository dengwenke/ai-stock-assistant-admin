# 管理后台设计规范（Admin）

## 1. 目标

- 保持管理端信息密度高、可读性强、操作路径短。
- 视觉风格与用户端同属一套产品语言，但管理端以效率优先。

## 2. 基础原则

- 颜色、间距、圆角优先使用全局 CSS 变量。
- 交互控件统一使用 Element Plus 组件，避免重复造轮子。
- 状态色语义固定：
  - 成功：绿色
  - 警告：橙色
  - 错误：红色
- 任务与调度信息优先以表格呈现，筛选区固定在表格上方。

## 3. 布局规范

- 外层壳使用 `src/layouts/AdminLayout.vue`。
- 标准页面结构：
  - 页面标题区
  - 筛选/操作区
  - 数据展示区（表格或卡片）
  - 分页区（如有）
- 页面内容区默认内边距保持与 `el-main` 一致，不单独设置过大边距。

## 4. 样式规范

- 全局样式入口：`src/styles/index.css`。
- 禁止在多个页面复制同一套颜色和阴影定义。
- 页面级 `scoped` 样式仅处理当前视图特有布局，不覆盖全局 token 语义。

## 5. 可维护性规范

- 菜单与路由一致：新增菜单时必须同步新增路由（反之亦然）。
- 视图层禁止直接拼接 API URL；统一在 `src/api/client.ts` 封装。
